## 토프링 7장 스프링 핵심 기술의 응용

- 기존에는 `XmlSqlService` 생성자에서 XML **언마샬링**작업을 했다
- XML에서 SQL 정보를 읽어오는 책임과 SQL들을 매핑하는 책임에 맞는 개별의 인터페이스를 추가했다
  - `SqlRegistry`, `SqlReader`
- 위 두 개의 인터페이스를 `XmlSqlService` 한 클래스에서 모두 구현하여 **자기 참조**를 하도록 리팩토링했다
- **자기 참조**로 되어있는 각 책임들을 분리하기 위하여 구현 클래스들을 추가했다 
  - `SqlRegistry`를 구현하는 `HashMapSqlRegistry` 추가
  - `SqlReader`를 구현하는 `JaxbXmlSqlReader`추가
- 위의 구현체들을 DI하도록 **디폴트 의존관계를 갖는 `DefaultSqlService`를 추가**
- 지금은 XML 정보를 읽어들이는 `JAXB`로 구체적인 기술과 API에 종속되어 있기 때문에 **서비스 추상화** 적용
  -  Spring의 `Jaxb2Marshaller` 테스트
- `BaseSqlService`와 `OxmSqlService`의 중복되는 메소드를 제거하기 위해 해당 메소드들은 `BaseSqlService`에게 실행하도록 위임했다
- **SQL을 조작하는 책임을 가진 `UpdatableSqlRegistry`인터페이스를 구현하는** 
  - `ConcurrentHashMapSqlRegistry` 클래스를 추가하였다
  - `EmbeddedDbSqlRegistry`클래스를 추가하였다
    - 내장형 디비는 H2를 사용, `update(Map<String, String> sqlmap)`에 트랜잭션을 적용하였다

```java
public class EmbeddedDbSqlRegistry implements UpdatableSqlRegistry {
    JdbcTemplate jdbc;
    TransactionTemplate template;

    public void setDataSource(DataSource dataSource) {
        jdbc = new JdbcTemplate(dataSource);
        // DataSource로 TransactionManager를 만들고 이를 이용해 TransactionTemplate을 생성한다.
        template = new TransactionTemplate(new DataSourceTransactionManager(dataSource));
    }
    ...
}
```

dataSource로 `JdbcTemplate`을 생성하여 쿼리를 실행시키는 것과 <br>
dataSource로 `TransactionManager`를 생성하고 `TransactionTemplate`을 생성하여 쿼리를 실행시키는 것의<br>
트랜잭션 동기화 저장소를 사용하는 차이점은 알겠지만 확실한 (각 클래스들의)차이점을 알진 못 하겠다.<br>
<br>

7장에서 개인적으로 SQL을 수정하는 기능을 추가하면서 `UpdatableSqlRegistry`인터페이스를 추가하고<br>
`SqlRegistry`를 추상화 한것이 인상깊었다.
- `ConcurrentHashMap`을 사용한 구현체와 `내장형 디비`를 사용한 구현체를 추가한 것

## 입사확정

스타트업 면접을 볼 기회가 생겼다. 개인적인 일정으로 인해 양해를 구하여 감사하게도 1차,2차 면접을 빠르게 볼 수 있었다.<br>
제안한 연봉을 승낙해주셨고 **다음 주 수요일로 입사일자를 정하였다.** 빠르게 안내해주셔서 감사하다.<br>

`B2B`회사이며, (개인적으로 느끼기엔) 메인은 특허와 판례정보에 대한 검색엔진서버를 제공하는 것 같다.<br>
- *사실 1차 면접보러 가면서도 긴가민가했다. 내가 원하고 잘 할수 있는 일이 회사에 기여가 될까 싶었다*
- *공고는 5년차 이상이였지만, 3년차가 지원하였는데도 불러주셔서 면접은 보고 싶었던 마음이 있었다*
<br>

1차 면접에서 검색엔진서버 외적으로 `MSA`를 도입중에 있다고 말씀해 주셨고 `테스트 자동화 구축`을 생각하고 계신다고 했다.<br>
2차 면접은 나에 대해 알고싶어하는 질문들이였어서 모두 솔직하게 대답했다.<br>
- *나에 대해 알고 싶어하는 상황이 재밌었다.*
- *학습하는 태도를 좋게 봐주신 것 같다.*
<br>

느낀점은 **사용자가 늘어남에 따라 품질에 신경쓰고 있다는 느낌이 들었고 업무 환경이나 면접도 좋다고 느꼇다.**<br>

첫 회사 입사할 때 느꼇던 그 설렘이 느껴졌다.<br>
이전에 다녔던 SI 회사와 굉장히 다를 것 같다<br>
1. 내부 팀원들은 어떤 분들이시고 어떤것에 관심이 있을까, 어떤 고민들을 하고 계실까
3. 서비스는 어떤 식으로 구성되어 있을까
4. 어떤 스타일로 개발하고 어떻게 의사결정을 할까
5. 분산환경은 어떻게 적용할까
6. (개발하고 납품하는게 아니라)서비스를 제공하는 회사는 어떤식으로 업무를 처리할까

**빨리 적응하고 분석해서 서비스에 기여하고 이전 회사에서 맞닥뜨리지 못했던 문제들을 경험하고 해결해보고싶다.**<br>

## 토프링 7장 마무리

처음에는 하나였던 빈 설정 클래스를 빈의 종류와 적용 환경 등을 고려해 여러 개로 분리 한 뒤, `@Import`로 모아 메인 설정 클래스만으로 설정이 가능하게 했고,<br>
프로파일을 적용해서 환경에 맞는 빈만 적용되게 했다. <br>

- `@SpringJUnitConfig` 는 JUnit Jupiter의 @ExtendWith(SpringExtension.class) 와 Spring TestContext Framework 의 @ContextConfiguration 을 **결합한 합성 주석**이다.
  - 예제에서 `@ContextConfiguration`을 제거하고 `@SpringJUnitConfig`로 수정했다
- `@Import`를 사용하여 `싪제 필요한 DI`**AppContext** 정보 중 `SQL 서비스용 빈`**SqlServiceContext** 설정 정보 분리
  - 자바 클래스로 된 설정 정보를 가져올 때는 `@ImportResource` 대신 `@Import`를 이용한다.
- `@Profile`과 `@ActiveProfiles` 적용
  - `@Profile`이 붙은 설정 클래스는 `@Import`로 가져오든 `@ContextConfiguration`에 직접 명시하든 상관없이 **현재 컨테이너의 활성 프로파일 목록**에 자신의 프로파일 이름이 들어있지 않으면 무시된다.  
  - **활성 프로파일** : 스프링 컨테이너를 실행할 때 추가로 지정해주는 속성
- `@PropertySource`와 `Environment`객체 사용하기
- `@Value` 프로퍼티 정보 사용하기
- `@Import(value = SqlServiceContext.class)` 이 메타 어노테이션을 가진 `@EnableSqlService`를 추가해서 **AppContext**에 적용하였다

온보딩도 체계가 갖춰져있다고 느껴졌고 **트레이닝 프로젝트**, **코드 리뷰**를 진행한다고 해서 설레이는 상태다.<br>
누구한테는 당연하겠지만 나에게는 `회사에서 개개인에게 시간과 비용을 투자해서 어느정도 성장을 기대한다`라는 것이 SI에서는 상상도 할 수 없는 환경이기 때문에.. 너무 좋았다<br>

**지금 존재하는 기술적 문제나 부채들을 처리하기 위해 노력하고 있고 서비스를 알리기 위해 노력하고 있다는 것을 보고 느낄 수 있었다.**<br>
`"스스로 열심히하면 폭발적인 성장을 할 수 있겠다. 빨리 서비스에 기여해서 내가 끼치는 영향을 보고싶다."`<br>

<br>

트레이닝 프로젝트가 완전히 새로운 언어와 프레임워크로 진행되기 때문에 부담은 크지만 열심히 해보자<br>

**특허법의 목적**  
발명을 `보호`, `장려`하고 `기술의 발전`을 촉진하여 산업발전에 이바지함이 목적  

**발명의 보호,장려**
1. **실체적 보호**
   - 특허를 받을 수 있는 권리에 의한 보호 : 특허부여청구권,보상금청구권,선사용권 등
   - 특허권에 의한 보호 : 침해금치청구권,손해배상청구권,신용회복청구권 등
2. **절차적 보호**
   1. 출원단계
      - 공지예외적용 주장,조약우선권,국내우선권,분할출원 등
   2. 심사단계
      - 우선심사제도,신청에 의한 출원공개제도 등
   3. 설정등록 후
      - 특허의 정정, 정정심판 등

**발명의 이용**  
1. **실시에 의한 이용**
   - 특허권자의 실시
   - 제 3자의 실시 : 특허권이 소멸된 경우
2. **공개에 의한 이용**
   - 기술문헌적 이용 : 중복투자 및 중복연구 방지, 개량 기술 개발 유도

**발명의 법률 규정**  
`특허법 제2조(정의)` : "발명"이라 함은 자연법칙을 이용한 기술적 사상의 창작으로서 고도한 것을 말한다.  
발명의 정의를 특허법에 규정할 것인가, 아니면 학설이나 판례에 위임할 것인지는 `입법정책적인 문제`임  
   - 특허법에 규정하면 법적 안정성을 도모할 수 있지만
   - 새로운 유형의 발명에 유연하게 대처하지 못하는 단점이 있다

**발명의 성립성 요건**  
1. 자연법칙의 이용
2. 기술적 사상
3. 창작성
4. 고도성

**발명의 성립성이 없는 예**
1. 비발명
   - 자연법칙 그 자체
   - 자연법칙에 위배된 것
   - 자연법칙을 이용하지 않은 것 : 경제법칙,게임의 규칙
   - 컴퓨터 프로그램 언어 자체
   - 반복하여 동일한 효과를 얻을 수 없는 것
   - 기타 정보의 단순한 제시, 미적 창작물, 기능, 발견 등
2. 미완성 발명
   - 과제해결수단이 완전하게 기재되어 있지 않은 경우
   - 과제해결수단은 제시되었으나 과제해결이 현저히 의심스러운 경우

**발명의 성립성이 문제되는 경우**  
1. 용도발명
   - 발견과 구별이된다.  
   - 특정한 물질 자체가 지니는 용도를 발견하고 새로운 용도로의 이용과정에서 창작적 요소가 존재하는 발명
2. 전자상거래 관련 발명 (BM 발명)
3. 미생물 관련 발명
   - 미생물 자체의 발명, 생산하는 방법 또는 이용하는 발명 등

**발명의 성립성 위반 시 취급**
1. 특허법 제29조 제1항 본문 위반
   - `발명의 성립성 요건을 위반`한 경우 산업상 이용가능성 위반으로 거절결정
2. 특허법 제42조 제3항 및 제4항 위반
   - `미완성 발명`의 경우 거절결정 가능
3. 특허법 제47조 제2항 위반
   - `미완성 발명을 완성발명으로 보정한 경우` 거절결정

# 특허 출원과 명세서

2. 특허출원 전 권리획득 가능성 검토 (`서지사항조사`, `특정기술문헌조사`)
   - 특허취득 가능성 판단 및 기술적 범위의 확인
   - 무용한 특허출원 지양
3. 특허분쟁 대비 증거자료 입수 (`양수도조사`,`권리소멸조사`,`유효성조사`,`무효자료조사`,`침해조사`...)
   - 자사의 실시기술에 대한 공지기술 확보
   - 타사 보유 특허조사로 특허분쟁 사전 예방
   - 침해가능 특허에 대한 회피설계 가능 여부 파악
   - 특허권 소멸 여부 확인
   
**선행기술검색 D/B**  
1. 키프리스
   - 한국 특허, 세계 특허
   - 한국특허정보원
2. 윕스
3. DELPHION
4. NRI
5. MICROPATENT

**키프리스 검색 연산자**  

1. AND
2. OR `+`
3. NOT
   - 입력한 키워드 2개 중 1개는 반드시 포함, 다른 1개는 포함하지 않는 검색
4. 구문 `" "`
   - 공란이 포함되고 연속적으로 기재된 구문을 검색
5. 인접배열 `^#`
   - 첫 번째 검색어와 두 번째 검색어의 거리가 #단어 이하로 떨어져 있는 구문을 검색

**특허출원 시 제출서류**  
1. 특허출원서
2. 명세서
   - 발명의 설명 : 발명의 명칭 + 기술분야 + 배경기술
   - 발명의 내용 : 해결하고자 하는과제 + 해결수단 + 발명의 효과
   - 도면의 간단한 설명
   - 발명을 실시하기 위한 구체적인 내용
   - 청구범위
     - 청구범위를 적지 않은 경우에는 특허출원일 부터 `1년 2개월`이 되는 날까지 명세서에 청구범위를 적어야 한다.
3. 도면
   - `발명의 실시예를 구체적으로 표시`
    - 물건 발명은 반드시 첨부
4. 요약서
   - `기술정보의 용도`, 제출을 의무화함


**외국어특허출원**  
특허출원인이 명세서 및 도면을 국어가 아닌 언어(영어)로 특허출원할 수 있도록 하는 제도  
1. 미국 : 모든 언어로 특허출원가능 **영어 번역문 제출**
2. EPO : 모든 언어로 특허출원가능 **EOP 공식언어(영어, 독어, 불어) 번역문 제출**
3. 일본 : 일본어 또는 영어로 특허출원 가능 **일어 번역문 제출**
4. 중국 : 중국어로만 특허출원

**출원서류의 제출**  
1. 서면에 의한 제출
   - 서면으로 작성하여 특허청에 방문하여 제출하거나 등기우편으로 제출하는 것
   - `도달주의` : 특허청 또는 도달된 날 부터 효력 발생 (특허권 및 특허관련)
   - `발신주의` : 우편물의 통신일부인에서 표시된 날이 분명한 경우에는 표시된 날, 불분명한 경우에는 우체국에 제출한 날 
2. 전자문서에 의한 제출
   - 서류를 정보통신망을 이용하여 제출하거나 기록매체에 수록하여 제출하는 것

**방식심사**  
1. 서류의 반려 (절차의 불수리)
   - 특허출원 등의 절차에 `보정하기 어려운 중대한 하자`가 있어 출원서나 청구서 등을 수리하지 않는 행정처분
2. 반려 사유
   1. 1건 마다 서면을 작성하지 아니한 경우
   2. 출원 또는 서류가 불명확한 경우
   3. 국어로 기재되지 아니한 경우
   4. 출원서에 명세서를 첨부하지 아니한 경우
   5. 기간을 경과하여 제출된 서류 등
3. 절차의 무효
   - 유효하게 성립된 절차에 대하여 흠결이 있다는 이유로 그 `절차의 효력을 소급적으로 상실`시키는 행정처분
4. 절차의 무효 사유
   - 행위 능력을 위반한 경우
     - `미성년자, 피한정후견인 또는 피성년후견인`이 법정대리인에 의하지 아니하고 특허에 관한 절차를 밟는 경우
     - 피한정후견인 : 정신적 제약으로 사무를 처리할 능력이 부족한 사람
     - 피성년후견인 : 정신적 제약으로 사무를 처리할 능력이 지속적으로 결여된 사람
   - 대리권의 범위를 위반한 경우
   - 방식 위반
   - 수수료 불납

# 특허 요건

## **특허를 받을 수 있는 발명**  

**거절이유**  
1. 주체적 요건
  - 외국인의 권리 능력
  - 특허를 받을 수 없는 자 또는 공동출원
1. 실체적 요건
2. 조약
3. 절차적 요건

**산업상 이용가능성**  
- `산업상 이용할 수 있는 발명`
- 최광의개념 : 유용하고 실용적인 기술에 속하는 모든 활동
- 파리조약 : 산업재산권은 최광의로 해석되어 공업 및 상업, 농업, 채취산업, 제조 또는 천연의 모든 상품에 대하여도 적용
- 통설 및 심사실무
- `이용가능성`이란?
 - 이용 : 특허법상 `실시`와 같은 의미
 - 이용가능성 : 장래이용가능성만 있으면 OK
 - **의료방법발명의 경우**
   - 인류의 생명과 건강에 대한 기여 떄문에 산업상 이용가능성이 없다고 봄

**신규성**  
- 발명의 내용이 `사회일반에 아직 알려지지 않은` 발명으로서 `객관적 창작성`이 있는 것
- **신규성 상실 사유**
  1. 공지된 발명 : 불특정이 널리 알 수 있는 상태
  2. 공연히 실시된 발명 : 공연 실시, 공연 (비밀상태가 해제된 것), 실시 (특허법 제2조 3호)
  3. 반포된 간행물에 게재된 발명 : 불특정인 열람할 수 있는 상태
  4. 전기통신회선을 통하여 공중이 이용 가능하게 된 발명 : 인터넷 , 공중게시판 , 이메일 등 포함
- **신규성 판단 기준**
  1. 객체적 기준
     - 특허청구범위의 청구항에 기재된 발명과 공지기술
  2. 시기적 기준
     - 특허출원 시간 (시각주의)
  3. 지역적 기준
     - 국내 또는 국외 (국제주의)
- **신규성 판단 방법**
  - 청구항에 기재된 발명과 공지기술이 동일성이 있는지 여부 판단
  - 독립항과 종속항 : 독립항에 신규성이 있으면 종속항도 신규성이 있음
  - 물건발명, 물건의 제법발명 및 용도발명

**진보성**  
- 특허출원 전에 그 발명이 속하는 기술분야에서 통상의 지식을 가진 자가 `공지기술로부터 용이하게 발명을 할 수 없는 것`
- `주관적 창작성`
- **진보성 판단 기준**
  1. 주체적 기준
     - 발명이 속하는 기술분야에서 통상의 지식을 가진자 `통상의 기술자`
  2. 객체적 기준
  3. 시기적 기준
     - 특허출원 시간 (시각주의)
  4. 지역적 기준
     - 통상의 기술자의 수준은 `국내를 기준`으로 함

**선출원주의**  
- 동일한 발명에 대하여 `가장 먼저 특허출원한 자`에게 특허를 허여하는 주의
- 1발명 1특허원칙
- **똑같은 날 출원의 경우**
 - 협의에 의하여 하나의 특허출원인만이 특허를 받을 수 있다
 - 협의가 이뤄지지 않는다면 어느 누구도 그 특허에 대하여 특허를 받을 수 없다

## **특허를 받을 수 없는 발명**

**불특허발명**  
- 특허법 제32조(특허를 받을 수 없는 발명) : 공공의 질서 또는 선량한 풍속을 문란하게 하거나 공중의 위생을 해할 염려가 있는 발명에 대하여 ... 특허를 받을 수 없다.
- 국가사회의 일반적인 이익, 사회의 일반적,도덕적 관념
- 출원공개의 대상이 되지 않는다

**구체적인 예**  
1. 공서양속 위반
   - 해를 미칠 염려가 있는 피임기구
   - 아편흡입기구, 위조지폐 제조기구 등
   - 유독 물질을 함유한 장난감, 유해물의 허용량을 초과하는 화장품 등
2. 공중의 위생을 해할 염려가 있는 발명
